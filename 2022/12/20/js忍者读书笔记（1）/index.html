前言

这个系列将会是我读《JavaScript 忍者秘籍（第二版）》的读书笔记。

第一章——无处不在的 JavaScript

JS 语言特点

JavaScript 与其他高级语言的一些根本不同包括：

  • 函数是一等功名（一级对象）
    函数和像其他对象一样使用。可以通过字面量创建，可以赋值给变量，可以作为函数的参数进行传递。可以作为返回值从函数中返回。

  • 闭包
    闭包证明了函数对于 JS 的重要性。

  • 基于圆形的面向对象

和其他语言一样的有用的特征：

  • 生成器，可以给予一次请求生成多次值的函数
  • Promise，控制异步代码
  • 代理，控制对特定对象的访问
  • 高级数组方法
  • Map（字典合集）和 Set（包含不重复项目合集）
  • 正则表达式
  • 模块

浏览器与 JS

对于前端来说，JS 的主要执行环境是浏览器，在浏览器环境中有三个重要的概念：

  • DOM
  • 事件
    大部分 JS 应用都是事件驱动的应用，这表示大部分代码执行在某个特殊事件响应的上下文中
  • 浏览器 API
    浏览器提供获取设备的信息、存储本地数据和与远程浏览器交互的 API

最佳实践

除了要精通 JS 这本语言以外，一个成熟的开发还应该掌握

  • 调试技巧
  • 测试
  • 性能分析

第二章——运行时的页面构建过程

Web 应用的生命周期包括：

  1. 页面构建
  2. 事件处理

页面构建

浏览器暴露给 JS 引擎的主要全局对象是 window 对象，它代表了包含一个页面的窗口。
全局 window 对象最重要的属性是 document，它代表了当前页面的 DOM。
全局 window 对象会存在于整个页面的生存期之间，它存储着所有 JS 变量。

JS 代码可以修改 DOM 结构，但是不能够选择和修改还没有被创建的节点，这就是吧script元素放在页面底部的原因。

事件处理

浏览器执行环境的核心思想： 同一时刻只能执行一个代码片段，即单线程执行模型。
因此需要事件队列来跟踪已经发生但是尚未处理的事件，这个跟踪过程不参与事件处理线程，因此是异步的。

事件包括：

  • 浏览器事件
  • 网络事件
  • 用户事件
  • 计时器事件

总结

以上内容从总体上介绍了 JS 语言的特点以及 JS 的主要运行环境——浏览器。同时也表明了 JS 和浏览器的互动关系。
