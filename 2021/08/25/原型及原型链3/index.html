class继承

上文的继承方式都是基于原型实现的，缺点是继承的实现需要编写大量的代码，并且需要正确实现原型，但其实有更简单的继承方法：

ES6引入了关键字class，class的目的就是让定义类更简单。

改写Dog（）

我们来回顾之前Dog的实现方法：

  function Dog(name, color){
     this.name = name;
     this.color = color;
   }
  Dog.prototype.type = "犬";
  Dog.prototype.eat = function(){
     alert("吃狗粮");
   };

如果要用class关键字来编写，可以写作：

  class Dog {
      constructor(name){
        this.name = name;
       }
      type = "犬";
      eat(){
         alert("吃狗粮"); 
       };
   };

这样编写的好处是避免了Dog.prototype.type这种分散的代码；

两个构造函数之间的继承

若改写之前Dog和Animal之间的继承会是什么样的呢？

  class Dog extends Animal {
      constructor(name, color){
          super(name);//用super调用父类的构造方法
          this.color = color;
       }
      type = "犬";
      eat(){
            alert("吃狗粮"); 
       };
   };

class继承与原型继承的区别

其实在实现方法上没有区别，只是在实现上class简化了原型链的代码。
