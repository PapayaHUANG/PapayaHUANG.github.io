JS原型及原型链(二)

2021-08-25

普通函数的原型链

  • 在第一部分盘点了构造函数及实例之间的关系，其实Javascript对每一个创建的对象都会设置一个原型，指向它的原型对象。
    例如，创建一个Array对象：
    var arr = [1, 2, 3]
    其原型链是：
    arr ---> Array.prototype --> Object.prototype --> null
  • Array.prototype定义了indexOf()、shift()等方法，因此可以在所有array对象上直接调用这些方法。
  • 当我们创建一个函数时：
    function foo(){
return 0;
}
    函数也有一个对象，它的原型链是：
    foo --> Function.prototype --> Object.prototype --> null

对象之间的继承

  • 若现在有一个“动物”对象的构造函数：
    function Animal(){
 this.species = "动物";
}
    以及一个“狗”对象的构造函数：
    function Dog(name, color){
 this.name = name;
 this.color = color;
}
    如何使”狗“继承”动物“呢？

构造函数绑定

  • 使用call或者apply的方法，将父对象的构造函数绑定在子对象上，即在子对象的构造函数中加一行：
    function Dog(name, color){

  Animal.apply(this, arguments);

  this.name = name;
  this. color = color;
} 
var dog1 = new Dog("果果", "黄色");
console.log(dog1.species); //动物

prototype模式

  • 如果“狗”的prototype对象，指向一个Animal实例，那么所有“狗”的实例，就能继承Animal了：
    Dog.prototype = new Animal();
Dog.prototype.constructor = Dog;
var dog1 = new Dog("果果", "黄色");
console.log(dog1.species); //动物
    代码第一行，我们将Dog的prototype对象指向一个Animal的实例；
    它相当于完全删除了prototype对象原先的值，然后赋予一个新值；
    此事，Dog.prototype.constructor指向Animal；
    更重要的是，每一个实例的constructor属性也默认调用prototype对象的constructor属性；
    这样会导致继承链的紊乱，因此必须手动纠正，将Dog.prototype对象的constructor值改为cat,也就是第二行的意思。

直接继承prototype

  • 这是对上一种方法的改进，可以讲Animal中不变的属性直接写入Animal.prototype中：
    function Animal(){};
Animal.prototype.species = "动物";
    然后将Dog的prototype对象指向Animal的prototype对象：
    Dog.prototype = Animal.prototype;
Dog.prototype.constructor = Dog;
var dog1 = new Dog("果果", "黄色");
console.log(dog1.species); //动物
  • 与上一种方法相比，这样做效率更高（不用执行和建立Animal实例），省内存，缺点是Dog.prototype和Animal.prototype都指向了同一个对象，那么任何对Dog.prototype的修改，都会反映到Animal.prototype:
    console.log(Animal.prototype.constructor); //Dog

    利用空对象作为中介

    var F = function(){};
F.prototype = Animal.prototype;
Dog.prototype = new F();
Dog.prototype.constructor = Dog;
    F是空对象，几乎不占内存，同时修改Dog.prototype是不会影响到Animal的prototype的。
    console.log(Animal.prototype.constructor); // Animal
    将上述方法封装成一个函数：
    function extend(Child, Parent){
   var F = fucntion(){};
   F.prototype = Parent.prototype;
   Child.prototype = new F();
   Child.prototype.constructor = Child;
   Child.uber = Parent.prototype;
}
    使用方法如下：
    extend(Dog, Animal);
var dog1 = new Dog("果果", "黄色");
console.log(dog1.species); //动物

拷贝继承

  • 即把父对象的所有属性和方法，拷贝进子对象：
    首先，还是把Animal的不变属性，都放在它的prototype对象上：
    function Animal(){};
Animail.prototype.species = "动物";
    然后再写一个实现拷贝目的的函数：
    function extend2(Child, Parent){
   var p = Parent.prototype;
   var c = Child.prototype;
   for ( var i in p ){
         c[i] = p [i];
   }
   c.uber = p;
};
extend2(Dog, Animal);
var dog1 = new Dog("果果", "黄色");
console.log(dog1.species); //动物
