CSS样式的继承

2021-04-21

前言

CSS 的全称为 Cascading Style Sheet，所以 Cascading（叠层）是理解 CSS 一个很基础和关键的概念。

所谓的“叠层”，就是说 CSS 设定的样式是可以叠加的，具体一点即：样式可以被继承，或者被覆盖。而叠层的顺序遵循一个重要的逻辑就是“子承父性”。

谁是子谁是父涉及到 HTML 标签的嵌套，可以打破 HTML 嵌套框架的方式是添加 class 或者 id 属性。

样式选择器分类

在使用 CSS 设定元素属性的时候，我们把以 HTML 标签为基础的元素命名为“元素选择器”；把 class 元素命名为“类选择器”；而 id 元素命名为“ID 选择器”。

在 CSS 处理叠层的优先级时，除了要考量以上三个选择器，还要把<style>属性内的声明纳入在内，则<style> > ID > 类 > 元素。

MDN 是用 4 位数来计算各种选择器的优先级的。MDN 原文

除此之外需要额外注意的点是，CSS 的继承不是子承父的所有属性，一般情况来说，如果样式会影响文本的外观会被继承，其他属性不能被继承，如 border。

要体会这一点，可以从变换子选择器的 all 属性值来体会。

举例

<blockquote>
  <p>This blockquote is styled</p>
</blockquote>

<blockquote class="fix-this">
  <p>This blockquote is not styled</p>
</blockquote>

blockquote {
  background-color: red;
  border: 2px solid green;
}
.fix-this {
  all: inherit;
}

在这里，类选择器被规定需要继承元素选择器的 CSS 属性。

但是从效果上来看，类选择器仅继承了边框，没有继承元素选择器的背景色和边框属性。
background-color这个属性不可以被继承，其默认属性是 transparent（透明），这样父属性的背景色才能被看见，更符合呈现逻辑。

而在这个例子中，两个选择器的原生关系并不是父元素和子元素的关系，是两个同等重要的 block 元素，所以即便fix-this被规定需要 inherit时，保留了边框，但是移除了一开始对边框属性的设定。
