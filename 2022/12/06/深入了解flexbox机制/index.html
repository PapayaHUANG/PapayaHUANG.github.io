深入理解Flexbox机制

2022-12-06

前言

今天看到了一篇讲解 flexbox 的文章，而我之前看到过从属性和属性的值这个角度来将 CSS 语法不大一样，觉得很有意思，所以记一下笔记。

为什么会出现 flexbox

CSS 的默认布局模式是“流”（flow），也就是 块级元素 以栈的方式布局，而 内联元素 在块的内部相互连接，就好像 Word 文档的布局一样。
Flexbox 的出现给这样的布局增加了更多的灵活性（flexibility）， 我们可以使用 Flexbox 来控制元素的分布和对齐方式、放大和缩小以及对额外空间的分配。

flexbox 的核心概念

flexbox 是通过两个轴来控制元素的分布和对齐方式的。

主轴(Primary Axis)和交叉轴(Cross Axis)

设置主轴和交叉轴的方向需要借助flex-direction属性。

  • 主轴
    默认情况下，元素会在主轴的起始点依次排开，也就是说主轴解决的是元素的分布问题(distribution)。
    在主轴上，是将所有元素作为一个整体来布局。

  • 交叉轴
    默认情况下，元素会在交叉轴方向填满整个容器，也就是说交叉轴解决的是元素的对齐问题(alignment)。
    在交叉轴上，是将每个元素作为一个个体来布局。

因此可以得出不同属性适用的范围：

  • justify
    是用来定位主轴上的元素

  • align
    是用来定位价差轴上的元素

  • content
    指定的是一组可以被分布的元素

  • items
    指定的是单个元素

也因为主轴和交叉轴有不一样的使命，所以：

  • justify-content（主轴分布）独有的值：
    space-between/ space-around / space-evenly

  • align-items（交叉轴对齐)独有的值：
    stretch / baseline

  • 两者共有的值
    flex-start / center / flex-end

这个时候就引出另外一个属性 —— align-self, 它的值和align-items一样，只不过它的适用范围是 _单个元素层面_，而之前的两个属性适用范围都是从 容器层面考虑的。

以上逻辑也解释了为啥没有justify-self。

假设大小(Hypothetical size)

在讨论 flexbox 种元素的放大缩小以及额外空间分配问题，先要引入一个概念假设大小。

在 flow 布局中，width是一个硬约束（hard constraint)， 也就是说即便我们设定了一个明显超大的值，也会尽量显示出来。
但是在 flex 布局中，width不再是一个硬约束，它变成了一个可供参考的值，也就是假设大小。

元素的放大与缩小

flex-basis

如果flex-direction为row， flex-basis就相当于width; 如果flex-direction为column， flex-basis就相当于height。

我们通过flex-basis来设置主轴上元素的假设大小。由于设置的是假设大小，所以这个值并不是硬约束。

flex-grow

一般来说当元素在主轴分布的默认状态会有额外的空间，这个时候就可以设定flex-grow来填充这个空间，flex-grow的值为 0 。
如果多个元素都设置了flex-grow值，则按比例分配空间。

flex-shrink

我们可以通过调整flex-shrink的值，使得容器在变小的时候，内部元素不等比缩小。
若想要元素不因为在缩小的时候变形，可以将元素的flex-shrink设置为0。
在缩小容器的时候要注意一个点，就是元素的最小尺寸，如一个输入框的最小尺寸为170px-200px,而一个文本的最小宽度为其中最长字符串的长度。所以即便我们把flex-shrink保持在默认值1，也就是说元素应该发生缩小，但是最终当容器小到一定程度的时候，输入框和文本会导致元素溢出。这个时候我们只需要将min-width修改为0就可以解决。
但是因为min-width：auto实际上是一个阈值，所以随意修改也会造成不必要的后果。

额外空间分配

gap

使用gap可以调节元素之间的值。

margin: auto

适用这个属性可以实现导航栏，logo 和菜单分置两端：

<style>
  ul {
    display: flex;
    gap: 12px;
  }
  li.logo {
    margin-right: auto;
  }
</style>

<nav>
  <ul>
    <li class="logo">
      <a href="/"> Corpatech </a>
    </li>
    <li>
      <a href=""> Mission </a>
    </li>
    <li>
      <a href=""> Contact </a>
    </li>
  </ul>
</nav>

wrapping

当我们将flex-wrap的值设置为wrap之后，容器中的主轴和交叉轴就编程多个，所以对对应的属性也应该重新思考。
