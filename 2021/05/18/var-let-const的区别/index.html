var/let/const的区别

2021-05-18

昨天在微信公众号看到一篇面试概念题的帖子，觉得很基础也很有意思，所以尝试自己分类，并收集资料解释一下。

var、let和const之间的区别

历史：

let和const是随着2015年ES6的诞生而产生的，在此之前声明变量只有var，let和const很好地补充了var的不足。

区别：

作用域

  • var

如果var在函数外声明，var的作用域为全局作用域；

如果var在函数内声明，var的作用域在该函数内。

  • let

块级作用域（block scoped），即{ }之间。

  • const

块级作用域

关于函数作用域和块级作用域的区别，可以查阅参考文章

变量更新和声明

  • var

var可以重复声明并更新。

  • let

let可以在同一作用域更新变量，但let不可以重复声明。

  • const

const不可以被更新，也不可以重复声明。

因此const声明时就一定要赋值。

但如果用const声明一个对象，对象的属性可以更新。

提升（hoisting）

  • var

var存在变量提升，可以在声明之前被调用，其值为undefined

  • let

let存在变量提升，但没有var造成的变量提升问题，因为let没有被自动赋值，所以如果在声明前调用let，系统会报错。

  • const

const存在变量提升，和let一样，没有自动赋值。

由于前文提到的作用域的区别，var很容易造成bug，这点能够很好的被let和const解决。
