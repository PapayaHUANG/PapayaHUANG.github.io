git学习笔记

2021-07-15

由于准备用hexo和github一起配合搭建个人网站，所以在开始之前简单学习了一下git的基本知识，在此做一个梳理。

Git的核心是分布式版本控制系统，与之相对应的概念是集中式版本控制系统。

分布式版本控制系统的优势在于不必联网工作，以及强大的分支管理。

Git的核心概念：

  1. 版本库/Repository
  • 在这个目录下的所有文件都可以被Git管理起来；

每个文件的修改、删除，Git都能跟踪，就像一个路径清晰的时光机，可以在任何时刻追踪历史，或者在将来某个时刻还原。

  1. 工作区/Working Directory
  • 在电脑中可以看到的目录

  1. 暂存区/Stage
  • 暂全区是理解git通过调整HEAD指针的位置来回顾和还原文件的关键，也是和他人协同工作的基础。

Git命令辨析

git restore/git checkout --文件名/git reset HEAD 文件名
  • git restore可以实现后两者的功能，后两者是文件出在不同的区域是的撤销与删除
git checkout --文件名/git checkout 分支名
  • 前者是撤销修改，后者是切换所在分支，后者可以用语义更明确的git switch来替代
