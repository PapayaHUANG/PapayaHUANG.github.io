补充@media查询

2021-04-22

前言

《Head First》引入@media 的讲解是针对网站需要不同的样式表这个案例。

读完之后我对两个信息点比较感兴趣：

  1. 在语法结构方面，@media除了有screen和print媒体形式还有没有其他的媒体形式？

  2. 针对不同设备（如电脑、平板、手机）的显示，有没有特定的@media设定值的模板？

针对这两个点，我 google 了一些资料，现汇总一下：

语法结构

@media media type and (condition: breakpoint) {
  //CSS rules
}

在语法结构中，media type除了screen表示不同屏幕，print表示打印格式之外，还有两个类型：

  • 一个是all，表示所有媒体类型；
  • 另一个是speech，表示需要把页面内容读出来。

若不标明media type，则@media的默认为all。

设定 breakpoint

不同设备的显示应该选择什么具体的设定值，有没有模板，这个问题可以简化为——
“如何设定@media的 breakpoint？”

回答这个问题有两个思路：

  1. 根据设备的不同来设定 breakpoint；

  2. 根据内容的不同来设定 breakpoint；

如果是根据设备来设定 breakpoint，情况会变得非常复杂，仅一个 iphone 就已经有 6、7、8、9、10、11、12，若要维护起来会非常困难。

相关的设定参数，可以参加这个博客——CSS-Tricks

所以更省事的方式应该是根据内容的不同来设定 breakpoint:

换句话说，就是设定width的min/max值，或者同时设定 min 和 max 值，来管理一个范围区间的设备。

如：

@media only screen and (min-width: 640px) {
}

或

@media only screen and(max-width: 1024px) and (min-width: 640px) {
}

那么接下来的问题就是

min 值/max 值的定义

如果主要的显示设备是手机，那么就可以从 min-width 开始，根据实际情况，逐渐增大；

如果这要的显示这杯是大屏幕，那么就可以从 max-width 开始，根据实际情况，逐渐缩小。

但具体用什么值的问题还没有回答，好在一些 CSS 框架给了回答
如 bootstrap 就有对 breakpoint 的设定——bootstrap。
