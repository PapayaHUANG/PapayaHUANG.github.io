创建自己的hexo主题（三）

2021-08-28

回顾

上次博文介绍了页面的细节，这次介绍整个文章的样式以及我在搭建个人网站遇到的困难和解决办法。

主题样式

hexo主题样式采用了stylus预处理器，在编写的简便性和代码组织上更方便。

样式的接入

样式还是添加到 <head>标签内，样式路径可以借助hexo自带的辅助函数官方文档，代码如下：

  <!-- add style -->
  <%- css('css/style') %>

样式的模块管理

在建立好了总样式文件style.styl之后，我们就可以分别引入样式了:

  @import "_variables"
  @import "_partial/header"
  @import "_partial/article"
  @import "_partial/sidebar"
  @import "_partial/about"
  @import "_partial/blogs"
  @import "_partial/content"
  @import "_partial/post-tags"
  @import "_partial/categories"

特殊样式

图标

因为我的内容里包括一些中文互联网知名的社交平台图标，所以这个主题的图标我使用了阿里巴巴矢量图库。
引用方式是用<i class>引入，方便语义化管理。

代码高亮

最新版本的hexo已经封装好了代码高亮的功能，可以直接在config文件中配置。
但是这里我是安装了prism_plugin，有更多主题可以配置。
安装好插件后，需在config文件中增加相应配置，详情可以看官方文档。
需要注意的是，大括号在prism_plugin原设定中没办法转义，需要自行找到插件src文件夹中的index.js补充：

  const map = {
    '&#39;': '\'',
    '&amp;': '&',
    '&gt;': '>',
    '&lt;': '<',
    '&quot;': '"',
    '&#123;': '{',//补充部分
    '&#125;': '}'//补充部分
  };

图床

博客内容和网站本身涉及到一些图片的使用，我的方式是用Github搭建好了图床，然后用Picgo上传，截图工具我用的Snip。

困难及解决方法

整个网站搭建一直是磕磕碰碰，我主要遇到三个方面的问题：

  1. 收集信息的方法。比方说搭建hexo是我第一次接触ejs和stylus，以及网站的渲染过程，有一些知识盲区，在寻求办法的过程中我发现最可靠的还是官方文档。
  2. 布局问题。为了让整个网站在移动端适配，我采用了媒体查询+flex弹性布局，同时也遇到了双栏布局如何让菜单栏宽度保持不变的困难。最后用bfc+float解决了这个问题。
  3. hexo自带函数的运用。由于一开始没有仔细阅读官方文档，导致我不清楚有些函数只能在特定页面成立，在页面设置上走了一些弯路。如我把index页面作为了网站封面，就只能放弃利用hexo自带函数搭建博文摘要页面的选项，用archieve来实现博文清单。
