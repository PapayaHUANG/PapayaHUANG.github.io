http和https的区别

2021-05-19

前言

最近在看前端面试的题库，第一道就是http和https的区别，感觉牛客网上写得虽然很详细很清晰，但是太冷冰冰，是好的背诵资料，但不是理解资料，所以我又搜了英文的资料，结合在一起，自己写一下回顾总结。

概念

  • HTTP

hypertext transfer protocol

超文本传输协议

是由Berners-Lee在90年代初创建的，HTTP这个网络协议标准帮助客户端和服务器端之间可以实现数据交换。

HTTP是无状态的（a stateless system), 主要专注于呈现信息，所以存在安全隐患。

HTTP是明文传输的（plaintext）。

  • HTTPS

HTTPS并不是HTTP的对立面，他是HTTP的小兄弟。他和HTTP一样，也是客户端与服务器端的数据传输协议。

但他更高级、更安全。

S表示Secure（安全），是由TLS（Transport Layer Security）实现的，TLS是SSL（Secure Sockets Layer）的升级版本，是一种安全协议，确保客户端和服务器端的数据是加密传输的。

TLS除了加密数据，还会验证服务器端，保护数据在传输的过程中不被动手脚。

HTTP更像是目的地，而SSL是通向目的地的路程，HTTP响应数据，呈现在显示屏上，而后者管理通向这个目的地的路程，使得过程更加安全。

区别

  • HTTP

不安全

TCP端口：80个

在应用层面（application layer)工作

不需要SSL证书

不需要域名验证（domain validation）

数据不加密

  • HTTPS

安全

TCP端口：443个

使用TLS

需要安装由CA（certification authority）认证的SSL证书

需要域名验证、证书以及法律文件验证

数据加密

HTTPS的优点

更安全

谷歌搜索排名更靠前

HTTPS的缺点

SSL证书是付费的

客户端运行速度更慢
