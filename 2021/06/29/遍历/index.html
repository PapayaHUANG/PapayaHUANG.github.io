遍历区分

2021-06-29

前言

最近在学react，在写component的时候会遇到需要遍历数组的情况，主要用到的是map()和filter()，刚好看到了总结遍历的文章，所以在此自己也动笔做一下笔记。

简单总结

  • forEach —— 只遍历数组，不返回新数组；
  • map —— 对每项元素做改变后，返回新数组；
  • reduce —— 对每项元素做叠加，返回叠加后的值；
  • some —— 判断数组中某些项是否符合条件（内部return true时跳出整个循环）；
  • every —— 判断数组中每一项是否符合条件 （内部return false时跳出整个循环）；
  • filter —— 筛选出符合条件的数组；
  • find —— 筛选数组：找元素；
  • findIndex —— 筛选数组：找索引；
  • includes —— 判断数组是否含有某值；

forEach（）批量操作

  • 遍历数组全部元素（自动遍历数组length次）
  • 无法break中途跳出循环，不支持return操作推出循环
  • 不产生新的数组

map（）映射

  • 对数组中的每一项通过某种计算，产生新的数组
  • 返回新数组，不改变原数组
  • 无法break中途跳出循环，不支持return操作推出循环
  • 使用return操作输出，会循环数组每一项，并在回调函数中操作
  • 回调函数参数：item/index/array

filter（）筛选过滤

  • 筛选出数组中符合条件的项，组成新数组
  • 创建新数组，不改变原数组
  • 输出的事判断为true的数组元素组成的新数组
  • 使用return操作输出，会循环数组每一项，并在回调函数中操作
  • 回调函数参数：item/index/array

some（）有符合

  • 检测数组中是否有某些符合条件，只要有一项符合了，马上return true，退出循环
  • 不创建新数组、不改变原数组
  • 当内部return true时退出循环
  • 使用return操作输出，会循环数组每一项，并在回调函数中操作
  • 回调函数参数：item/index/array

every（）全符合

  • 检测数组中的每一项是否符合条件，只要不符合就返回false，退出循环
  • 不创建新数组，不改变原数组
  • 当return false时退出循环
  • 使用return操作输出，会循环数组每一项，并在回调函数中操作
  • 回调函数参数：item/index/array

find（）返回一个符合的

  • 找元素（找出某个符合条件的元素）
  • 不创建新数组，不改变原数组
  • 输出的一旦判断为true则跳出循环输出符合条件的数组元素
  • 使用return操作输出，会循环数组每一项，并在回调函数中操作
  • 回调函数参数，item/index/array

findIndex（）返回序号

  • 找索引（找出某个符合条件的元素的索引）
  • 其他特点和find（）相同

reduce（）递归累加

  • 不一定是数学意义上的叠加计算，这里的叠加指：让数组中的前项和后项做某种计算，并累计最终值
  • 创建新数组，不改变原数组
  • 无法break中途跳出循环，不支持return操作推出循环
  • 输出新数组
    reduce()方法接收一个回调函数作为第一个参数，回调函数又接受四个参数，分别是；
  1. previousValue =>初始值或上一次回调函数叠加的值；
  2. currentValue => 本次回调（循环）将要执行的值；
  3. index=>“currentValue”的索引值；
  4. arr => 数组本身；
    reduce()方法返回的是最后一次调用回调函数的返回值；

includes（）判断是否有（完全匹配）

  • 只是判断数组是否含有某值，不用return，不用回调函数，输出true或false
