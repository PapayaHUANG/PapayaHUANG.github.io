JS原型及原型链(一)

2021-08-21

前言

因为总结了javascript计算器项目过程，且javascript计算器主要是通过class原型继承来实现的，所以在这篇文章里我尝试理解一下原型及原型链。

从实例对象到构造函数

原始模式

Javascript是一种基于对象（object-based）的语言。
假定我们把狗看成一个对象，它有“名字”和“颜色”两个属性。

  var Dog = {
    name: '',
    color: ' '
   }

再根据这个格式，生成两个实例对象。

  var dog1 = {};
    dog1.name = "果果";
    dog1.color = "黄色";
  var dog2 = {};
    dog2.name = "汤圆";
    dog2.color = "黑色";

这样就把两个属性分别封装到一个对象里，但是这样做有两个问题：

  1. 如果实例的数量变多，写起来就非常麻烦；
  2. 实例和原型之间没有任何方法和联系。

原始模式的改进

可以通过一个函数来解决上述代码重复的问题：

  function Dog(name, color){
    return {
      name: name,
      color: color
    }
  }

然后通过调用函数的方式来生成实例对象：

  var dog1 = Dog("果果", "黄色");
  var dog2 = Dog("汤圆", "黑色");
  • 这样的写法问题是，dog1和dog2之间没有内在联系，不能反映出他们是同一个原型对象的实例。

构造函数模式

为了解决从原型对象生成实例的问题，javascript提供了一个构造函数模式。

构造函数其实就是一个普通函数，但是内部使用了this变量。对构造函数使用new运算符，就能生成实例，并且this变量会绑定在实例对象上。

比如，狗的原型对象可以这样写：

  function Dog(name, color){
    this.name = name;
    this.color = color;
   }

现在我们就可以生成实例对象了：

  var dog1 = new Dog("果果", "黄色");
  var dog2 = new Dog("汤圆", "黑色");
  console.log(dog1.name); //果果
  console.log(dog1.color); //黄色

这时dog1和dog2会自动含有一个constructor属性，指向他们的构造函数：

  console.log(dog1.constructor === Dog); //true
  console.log(dog2.constructor === Dog); //true

Javascript还提供了一个instanceof运算符，验证原型对象与实例对象之间的关系：

  console.log(dog1 instanceof Dog); //true
  console.log(cat2 instanceof Cat); //true

构造函数的问题

构造函数存在内存浪费的问题
我们现在为Dog对象添加一个不变的属性type，再添加一个方法eat()，那么原型对象Dog就变成了下面这样：

  function Dog(name, color){
    this.name = name;
    this.color = color;
    this.type = "犬"；
    this.eat = function(){
       alert("吃狗粮");
     };
  }

还是使用同样的方法生成实例：

  var dog1 = new Dog("果果", "黄色");
  var dog2 = new Dog("汤圆", "黑色");
  console.log(dog1.type); // 犬
  dog1.eat(); //吃狗粮

这样做一个很大的弊端就是，对于每一个实例对象，type属性和eat()方法都是一模一样的内容，每一次生成一个实例，都必须为重复的内容，占用内存。

  alert(dog1.eat == dog2.eat); //false

Protoype模式

为了解决上述的重复和占用内存的问题，可以使用prototype模式，将所有功能相同的实例都指向某个内存地址。

Javascript规定，每个构造函数都有一个prototype属性，指向另一个对象。这个对象的所有属性和方法，都会被构造函数的实例继承。

这样上述例子就可以修改成：

  function Dog(name, color){
    this.name = name;
    this.color = color;
   }
  Dog.prototype.type = "犬";
  Dog.prototype.eat = function(){
      alert("吃狗粮");
   };

然后生成实例：

  var dog1 = new Dog("果果", "黄色");
  var dog2 = new Dog("汤圆", "黑色");
  console.log(dog1.type); // 犬
  dog1.eat(); //吃狗粮

这时所有实例的type和eat()方法，其实都指向同一个内存地址，指向prototype对象，这样就提高了运行效率。

  console.log(dog1.eat == dog2.eat); //true

Protype模式的验证方法

  • isPrototypeOf()
    这个方法用来判断，某个prototype对象和某个实例之间的关系。
    console.log(Dog.prototype.isPrototypeOf(dog1)); //true
console.log(Dog.prototype.isPrototypeOf(dog2)); //true
  • hasOwnProperty()
    每个实例对象都有一个hasOwnProperty()方法，用来判断某个属性到底是本地属性，还是继承prototype对象的属性。
    console.log(dog1.hasOwnProperty("name")); // true
console.log(dog1.hasOwnProperty("type")); // false
  • in运算符
    in运算符可以用来判断某个实例是否含有某个属性，不管是不是本地属性
    console.log("name" in dog1); //true
console.log("type" in dog1); //true
    in运算符还可以用来遍历某个对象的所有属性
