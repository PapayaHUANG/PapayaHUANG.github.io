CSS字体之@import和@font-face的区别

2021-04-21

前言

@import和@font-face都是 CSS 的内置规则，@import和@font-face都可以为网页添加 web 字体。但两个内置规则的实现方式有所不同。

@font-face

句法结构

@font-face {
  font-family: 'You can define the name of the font here';

  src: url('font url');
}

实现方式

浏览器从 WEB 字体文件所在的服务器下载字体，@font-face规则告诉浏览器需要加载src,url()内的指定文字字体，一旦加载，会把这个字体分配到font-family属性中指定的名字所在的选择器所指。

@import

句法结构

@import url（ 'font-url' ）;

@import规则必须放置在样式表顶部，位于其他任何规则之上。

实现方式

根据 URL 的地址，导入其他 CSS stylesheet（样式表）。

使用 web 字体的好处

可以在审美上让字体更加多样化，可以呈现出用户设备字体库没有的字体。

使用 web 字体的坏处

增加了页面加载的时长，增加了字体管理的复杂度，web 字体的设备兼容性不行。
