什么是异步JavaScript/JS中的同步和异步

在解决这个问题之前，要先解决何为异步。
基本上计算语言都是单线程（single-threaded）的，也就是一个接一个的任务排队运行，而异步用一句话概括就是几个任务同时运行。

什么时候会用到异步？

当从外部设备抓取（fetch）资源的时候，如：
从网络上下载一个文件；
向数据库接收以及发送数据、从网络相机获取视频流等。

Callback（回调）在js中的运用

btn.addEventListener('click', () => {
  alert('You clicked me!');

  let pElem = document.createElement('p');
  pElem.textContent = 'This is a newly-added paragraph.';
  document.body.appendChild(pElem);
});

这里的addEventListener第二个参数就是典型的callback；
setTimeout的第一个参数也使用了回调函数。

什么时候使用回调：

把一个复杂的任务分解成更小的步骤；
根据时间或顺序把这些步骤之间建立联系。

  • 使用回调的问题：
    由于回调会产生嵌套，太多回调就会产生回调地狱。

此时，promise就登场了。

promise在js中的运用

fetch('products.json').then(function(response) {
  return response.json();
}).then(function(json) {
  let products = json;
  initialize(products);
}).catch(function(err) {
  console.log('Fetch problem: ' + err.message);
});

fetch（）就是一个典型的promise的例子
