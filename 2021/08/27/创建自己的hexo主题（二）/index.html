创建自己的hexo主题（二）

2021-08-27

回顾

在上一篇博文里介绍了如何把一个主题的布局框架和页面框架搭建出来，这篇笔记里会介绍我是如何处理每个页面的细节。

主题配置文件

上篇博文提到过hexo的工作原理是会提取config.yml文件中的变量，动态渲染出最后的静态页面，所以在搭建页面细节时，可以利用config.yml中的变量，来更高效地搭建结构相同的页面细节：

在sidebar中，菜单栏的布局是一样，只是链接名称和对应路径不一样，这里就可以用到config：
首先在/theme/_config.yml添加如下文：

  nav:
    Home: /
    About: /about
    Categories: /categories
    Archives: /archives

然后在sidebar.ejs文件中引用nav变量，并枚举nav变量中的对象：

  <div class="menu">
    <% for (var i in theme.nav) { %>
        <a class="menu-item" href="<%- url_for(theme.nav[i]) %>">
        <%= i %>
        <a>
    <% } %>
  </div>

变量参数表可以看 hexo官方文档变量

Hexo常用变量

hexo自带的变量在主题的模块化起到了非常关键的作用，这里我想列举一下我在这个主题的搭建中比较常用的几个变量：

变量名 介绍 类型
site 全站信息 object
page 当前页面信息 object
theme 主题内config文件内参数 object
config 网站参数，主题外config文件 object
site.posts 全站博文 array of post objects
site.categories 全站分类 array
site.tags 全站标签 array
page.prev 上一个页面
page.next 下一个页面

特殊页面处理

归档页（archive）

hexo默认首页是index页面，并且index是用来放文章摘要的。这里的默认在内置的变量上是有相关设计的，如针对index有一些page相关的变量，方便博客文章逐条分页呈现，但是这些page相关的变量只在页面为index时生效；

这样就给我的网站搭建造成了问题，因为我需要把博文列表放在非index页面。但是好在有另外一个默认呈现博文列表的页面——archive，所以在我的主题充分利用了archive.ejs这个页面：

  <div class="archive-page">
      <div class="archive-title">#Posts</div>
      <div class="archive-block">
          <% page.posts.each(function(item){ %>
          <%- partial('_partial/article-archive', {item: item}) %>
          <% }); %>
          <%- partial('_partial/pagination') %>
      </div>
  </div>

注意此处{item: item}，因为需要把item这个参数传递给article-archive这个模块使用，所以会出现。

归档详细布局（article-archive）

我希望我归档页显示博文的标题和时间，并且点击标题可以看到完整文章，通过结合hexo变量实现：

  <div class="archive-post">
      <!-- title -->
      <h2 class="archive-post-title-archive">
          <a href = "<%- config.root %><%- item.path %>">
              <%- item.title || item.link %>
          </a>
      </h2>
      <!-- date -->
      <p class="archive-post-meta">
          <%= item.date.format(config.date_format) %>
      </p>
  </div>

翻页（pagination）

因为文章清单涉及到翻页，所以有必要设置每页显示的文章数和翻页按钮
每页显示文章数需要在主题外的config文件中设置：

  # Pagination
  ## Set per_page to 0 to disable pagination
  per_page: 5
  pagination_dir: page

而翻页按钮可以利用ejs的模块化来写专门的pagination.ejs

  <div>
     <ul class="paper">
        <% if(page.prev){ %>
        <li><a href="<%- config.root %><%- page.prev_link %>">上一页
        </a></li>
        <% } %> 
        <% if(page.next){ %>
        <li><a href="<%- config.root %><%- page.next_link %>">下一页
        </a></li>
        <% } %>
    </ul>
  </div>

文章布局

每篇文章我希望可以显示标题、时间、文章详情、文章的标签和分类，这些也可以通过hexo变量来实现：
显示文章标题：

  <h2 class="blog-post-title">
    <a href="<%- config.root %><%- item.path %>">
        <%- item.title || item.link %>
    </a>
  </h2>

显示文章时间：

  <p class="blog-post-meta">
    <%= item.date.format(config.date_format) %> 
  </p>

显示全文：

  <div class="post-content">
    <%- item.content %>
  </div>

显示归档

  <% if(item.categories && item.categories.length){ %>
    <% 
      var cats = [];
      item.categories.forEach(function(cat){
        cats.push('<a href="' + config.root + cat.path + '">' + cat.name + '</a>');
        });
    %> 
    <span class="post-categories">
       <i class='icon iconfont icon-icon-goodscategory'></i>
       <%- cats.join(', ') %>
    </span>
  <% } %>

显示标签

  <% if(item.tags && item.tags.length){ %> 
    <% 
      var tags=[];
      item.tags.forEach(function(tag){ tags.push('<a href="' + config.root + tag.path + '">' + tag.name + '</a>'); 
      });
    %>
     <span class="post-tags">
       <i class="icon iconfont icon-tag"></i>
       <%- tags.join('') %>
    </span>
  <% } %>

注意：因为tags和archives都可以回溯到archive页面，所以没有为标签和归档创建多余的页面。hexo文档对应内容
