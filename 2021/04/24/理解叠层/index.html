理解“层叠”（CADSCADING）的意义

理解层叠是如何工作的，可以帮助我们写出更容易维护的代码，也是 debug 的一个基础知识。

“层叠”是怎么工作的？

在给定的一组样式表中，浏览器以层叠的方式来确定具体使用哪一种样式。

那么“层叠”的方式具体是怎么工作的呢，其实可以把“层叠”看作一个对不同 CSS 声明的优先级的算法，对不同的 CSS 声明，“层叠”有一套加权逻辑来对 CSS 声明进行排序，最终通过确定不同 CSS 声明的重要性来决定浏览器样式的呈现。

“层叠”的排序规则是什么样的？

  • 样式的来源及重要性（指是否有!important声明）

  • 选择器的特定性（specificity）

  • 声明的顺序

  • 默认属性或继承属性

以上分类的重要程度由上到下，逐渐变小。

_其中需要更细化说明的有_：

  • 样式的来源及重要性（指是否有!important 声明）

样式的来源指的是声明来自浏览器（user-agent）、编写代码的程序员(author)还是用户(user)。

  • 来源和!important 混合后的重要性从高到低排序如下：
  1. user-agent and !important
  2. user and !important
  3. author and !important
  4. CSS animations, @keyframes（其实也属于 author）
  5. author
  6. user
  7. user-agent
  • 选择器的特定性
  1. 重要性从高到低排列如下
  2. 内联样式
  3. ID 选择器
  4. 类/伪类选择器
  5. 元素选择器/伪元素
