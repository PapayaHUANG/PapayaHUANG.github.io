《JavaScript面向对象精要》读书笔记（一）

2022-11-15

前言

这本书的作者是 Nicolas C. Zakas，也是大名鼎鼎红宝书的作者。我始终认为读书是最好的系统性吸收知识的方式，而做笔记可以强化吸收，所以有了这篇读书笔记。

在这篇笔记中，我会按照书本来的脉络，进行一些摘要。

注： 由于这本书写于 2015 年，所以核心 JS 还是停留在 ES6 及以前。

原始类型和引用类型

原始类型

  • boolean
  • number
  • string
  • null
  • undefined

注：现在 JS 中的原始类型还新增了：bigint, symbol

原始类型变量直接保存原始值，所以鉴别原始类型的最佳方法是直接使用关键字typeof。

但是typeof null会返回object，所以判断一个值是否为空的话使用console.log(value === null)更好。

引用类型

  • object
    引用类型保存引用对象的指针。

函数是最容易鉴别的引用类型，因为使用关键字typeof的返回值是funtion；其他引用类型可以使用instanceof来鉴别；鉴别数组可以使用Array.isArray()方法。

原始封装类型

  • string
  • number
  • boolean

它们虽然是原始类型，但是具备封装好的方法，可以直接使用，实现原理如下：

var name = 'Nicholas';
var firstChar = name.charAt(0);
console.log(firstChar); //"N"

等价于

var name = 'Nicholas';
var temp = new String(name);
var firstChar = temp.charAt(0);
temp = null; //对象的垃圾回收
console.log(firstChar); //"N"

函数

函数其实就是对象。
函数与其他对象不同的决定性特点是：函数存在一个被称为[[call]]的内部属性。
内部属性无法被代码访问而是定义了代码执行时的行为。

函数声明和函数表达式

  • 函数声明
function add(num1, num2) {
  return num1 + num2;
}
  • 函数表达式
const add = function (num1, num2) {
  return num1 + num2;
};

区别：函数声明会被提升到所在上下文的顶部，也就是可以先使用后声明。但是函数表达式不行。

重载

大多数面向对象的语言支持函数重载，它能让一个函数具有多个签名。
函数签名是由函数名字、参数的个数及其类型组成的。但 JavaScript 根据实际传入的参数决定带哦用函数的哪个版本，也就说 JavaScript 不存在重载，但是可以模仿这个行为：

function sayMessage(message){
  if(arguments.length === 0){
    message = "Default message"'
  }
  console.log(message)
}

理解对象

这部分主要讲对象的属性。可以将对象的属性看作键值对，将对象是为属性的哈希表来帮助理解。

属性有两种类型：

  • 数据属性
  • 访问器属性

数据属性

数据属性保存值，可以读写。所以相较于访问器属性，它具有[[Value]]和[[Writable]]，两个访问器属性不具备的特征。

访问器属性

访问器属性不保存值，而是用 getter 和 setter 来进行指定操作，因此它具有[[Get]]和[[Set]]这两个数据属性不具备的特征。

锁定对象属性

  • Object.preventExtension()
    这样对象的[[Extensible]]值就为false，不可逆。

  • Object.seal()
    这样对象的[[Extensible]]值为false，并且只能读写它已有的属性。不可逆。

  • Object.freeze()
    这样对象的[[Extensible]]值为false，并且数据属性都为只读。不可逆。
