理解BEM

2021-09-27

前言

在浏览MDN教程的时候喷到了BEM的概念，在此做一个总结。

定义

BEM的全称是Block，Element，Modifer，即块元素编辑器。是一种命名的方法，可以帮助程序员创建可以复用的前端组件和代码。

以下是使用BEM命名CSS的例子：

  /* Block component */
  .btn {}

  /* Element that depends upon the block */
  .btn_price {}

  /* Modifier that changes the style of the block */
  .btn--white {}
  .btn--small {}

Block

在这里block是顶层的组件，如例子中的按钮，可以被看作其他元素的父元素。

Elements

在block名称后加上下划线再接上具体元素的名称就是Elements，相当于顶层组件的子元素。

Modifers

顾名思义，modifer的意义是可以以组件的模式来批量更改元素，如设置不同的主题，Modifer的格式是block名称后加上双横杠再接上编辑器的名称。

与上述例子相关的html代码为：

  <a class="btn btn--white btn--small" href="#">
    <span class="btn_price">$99.99</span>
    <span class="btn_text">Subscribe</span>
  </a>

BEM的特性

有上述例子可以总结出BEM的特性：

  • 易用性 命名规则清晰；
  • 单元性 代码单元化可复用；
  • 灵活性 可以按照自己的方式来组织和配置单元。

使用BEM的原因

  1. modifer使得css的结构更加清晰，在给组件编辑样式的时候，可以从modifer着手，看是否需要编写新的css样式；
  2. BEM在标记语言中更易识别，可以通过BEM更快了解到元素之间的父子关系；
  3. 同一的命名规则更方便团队协作。
  4. BEM使得代码的结构更扁平，选择器类别分明，这样就避免了对选择器权重的纠结。
