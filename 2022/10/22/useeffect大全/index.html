理解useEffect

2022-10-22

前言

我自己学习 react 是直接从函数式组件开始的，所以在学之前并没有类组件的组件生命周期等思维的影响。但是由于对纯函数以及副作用等概念也不是特别清楚，所以在使用useEffect的时候还是会遇到一些问题，并且没办法判断出现问题的根源，所以想通过这篇文章梳理一下我自己学习useEffect的一些总结。

函数式组件的渲染行为

假设有这样一个组件：

function Counter(){
  const [count, setCount] = useState(0);

  function handleAlertClick(){
    setTimeout(()=>{
      alert('You clicked on: ' + count);
    }, 3000);
  }

  return(
      <div>
      <p>You clicked {count} times</p>
      <button onClick={()=>setCount(count + 1)}>
      Click me
      </button>
      <button onClick={handleAlertClick}>
      Show alert
      </button>
      <div>
  )
}

假设你先点击Click me三下，然后点一下Show alert,然后在回调函数setTimeout触发之前，再点击两下Click me。
弹出的对话框是 3 还是 5 呢。

最终答案是 3，而不是 5。

在解释为什么会生成这个结果之前，可以看一个 vanilla JS 的例子：

function sayHi(person) {
  const name = person.name;
  setTimeout(() => {
    alert('Hello, ' + name);
  }, 3000);
}

let someone = { name: 'William' };
sayHi(someone);

someone = { name: 'Billy' };
sayHi(someone);

some = { name: 'Patrick' };
sayHi(someone);

在这个例子中，弹出的对话框会每个三秒，分别按顺序弹出Hello, William, Hello, Billy,Hello, Patrick。因为每次调用 sayHi，都是调用传入的不同someone实参的name属性的值，即便这个调用要在 3 秒之后执行，绑定的值在 3 秒之后不会发生改变，所以生成了三个不同的名字。

那么类比我们上面 React 组件的例子，count值是一个在每一次调用组件函数的时候都是一个恒定的常量（constant），这个函数组件的每一次渲染，都相当于调用了一次这个组件函数，每一次调用就会是一个独一无二的count的值（state）。所以每次当我们点击三次Click me时，当时的count值为3, 而在我们点击Show alert时，传入回调函数的count值为3, 之后点击的Click me属于另外两次渲染，分别生成了另外两次count值，于我们记录的这次无关，所以最后结果是3不是5。

由这个例子可以得出： 在每次渲染中，props和state保持不变。（💡：这是不是和纯函数相关了！）

函数式组件渲染和副作用

我们把上面的例子稍微修改一下，增加一个useEffect:

function Counter() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    document.title = `You clicked ${count} times`;
  });

  return (
    <div>
      <p>You clicked {count} times </p>
      <button onClick={() => setCount(count + 1)}>Click me</button>
    </div>
  );
}

这里添加的副作用就是，改变网页的标题，并用useEffect来实现。
那么在这个例子中，useEffect是如何获取count的值的呢？

  • React： 呈现一个 state 值为0的 UI
  • 函数组件：
    • 渲染结果是`

      You clicked 0 times

      ;
    • 记得渲染完毕后要执行： ()=>{document.title = 'You clicked 0 times'}
  • React： 好，更新 UI，向 DOM 添加新的内容
  • 浏览器： 渲染！
  • React： 执行副作用：执行 ()=>{document.title = 'You clicked 0 times'}

也就是说，我们可以把组件渲染的逻辑也沿用到副作用上，每一次组件的渲染都会有自己的一个副作用。

总结： 每一次组件渲染中的所有函数（事件处理器、副作用、定时器或者 API 调用等）都捕获调用时定义的 props 和 state。

函数式组件的渲染逻辑

使用函数式组件需要完全摒弃掉组件生命周期的思维模式，取而代之的是：React 根据当前的 props 和 state 来 同步 DOM。

所以对于useEffect来说，它根据当前的 props 和 state，同步 React tree 以外的内容。

所以在这个思维模式下，“结果比过程更为重要”。

使用依赖数组避免不必要的渲染

基本规则是，如果在依赖数组中生命了变量，仅在变量发生变化的时候，useEffect才调用，如果依赖数组是空数组的话，仅在组件首次渲染的时候调用useEffect.

基本原则是：任何组件中的值，如果你在 useEffect 中使用了，就必须声明在依赖数组中。

减少依赖数组的使用

  • 使用函数式更新
useEffect(() => {
  const id = setInterval(() => {
    setCount((count) => count + 1);
  }, 1000);
  return () => clearInterval(id);
}, []);

在这里，useEffect 并不需要count的值来更新，因为每一次setCount都可以自给自足地累加，而每一次count的值也会自动发生变化，useEffect 只用调用当前一次。

  • 将数据更新和描述行为分离
    上述方法如果只有一个 state 还可以实现，但是遇到两个及以上 state 就难以实现：
function Counter() {
  const [count, setCount] = useState(0);
  const [step, setStep] = useState(1);

  useEffect(() => {
    const id = setInterval(() => {
      setCount((c) => c + step);
    }, 1000);
    return () => clearInterval(id);
  }, [step]);

  return(
    <>
    <h1>{count}</h1>
    <input value={step} onChange = { e => setStep(Number(e.target.value))}>
    </>
  )
}

以上的写法可以替换成：


  const [state, dispatch] = useReducer(reducer, initialState);
  const {count, step} = state;

  useEffect(()=>{
    const id = setInterval(()=>{
      dispatch({type: 'tick});
    },1000)

    return ()=>clearInterval(id)
  },[dispatch])


  const initialState = {
    count: 0,
    step: 1,
  };

  function reducer(state, action){
    const {count, step} = state;
    if(action.type === 'tick'){
      return {count: count + step, step};
    }else if(action.type === 'step'){
      return {count, step: action.step}
    }else{
      throw new Error()
    }
  }

使用reducer可以将数据的更新和数据行为分离，这样我们就不需要把step传入作为useEffect的依赖项，虽然我们传入了dispatch,但是在这个例子中，dispatch的值是保持不变的。同时在编写代码的时候，我们只需要关注 useEffect 内部需要的是什么行为。

  • 函数作为依赖项
    一个常见的错误是认为函数不可以作为依赖项，其实不然。
    认为不能将函数作为依赖项是因为声明在函数组件内部的函数在每一次渲染的时候都会发生变化，这样变化得过于频繁。
    可以使用useCallback打包这个函数：
function SearchResults() {
  const getFetchURL = useCallback((query) => {
    return 'https://...query =' + query;
  }, []);

  useEffect(() => {
    const url = getFetchURL('react');
  }, [getFetchUrl]);

  useEffect(() => {
    const url = getFetchURL('redux');
  }, [getFetchUrl]);
}

甚至我们可以进一步调整为：

function SearchResults() {
  const [query, setQuery] = useState('react')

  const getFetchURL = useCallback((query) => {
    return 'https://...query =' + query;
  }, [query]);

  useEffect(() => {
    const url = getFetchURL('react');
  }, [getFetchUrl]);
