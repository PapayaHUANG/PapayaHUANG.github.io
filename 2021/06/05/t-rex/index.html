T-rex模仿练习

2021-06-05

项目效果：

项目介绍：

  • T-rex就是google搜索主页在非联网状态google搜索框上方的小恐龙游戏。

  • 在这个项目练习当中我将使用vanilla js实现t-rex的核心功能。

项目实现目标：

  1. 画面包含：记分板、小恐龙、背景、作为障碍的仙人掌；

  2. 动态效果实现：

  • 记分板实时更新分数，

  • 小恐龙可以通过空格键控制跳跃，

  • 背景会循环向左滚动，

  • 仙人掌会随机生成并向左移动。

项目实现运用的js方法：

  1. appendChild()；

  2. setInterval();

  3. setTimeout();

  4. 针对html5的canvas和js结合实现背景无限循环滚动；

代码基本思路：

这个小游戏主要需要解决两个问题：

  1. 目标元素的移动动画，这个主要用setInterval以及setTimeout来反复调用同一个函数来实现；

  2. 函数分类，一部分是在游戏进行时运行，一部分是在游戏结束时运行；以及引发游戏结束的函数。

把第一个问题可以分解成四个小问题：

  1. 记分板的动画：
  • 我需要每秒实时更新记分板的数字，所以用setInterval（）来实现，func参数写数字的增加，delay参数写“1000”刚好是1秒；
  1. 小恐龙的跳跃：
  • 小恐龙的跳跃分成向上和向下两个函数，两个函数分别用setInterval来实现；
  1. 背景无限向左移动：
  • 这个的实现稍微复杂一些，我会在难点中详细说明；
  1. 作为障碍的仙人掌：
  • 我需要仙人掌在固定点随机时间生成，这里会结合setTimeout()和appendChild()，然后再使生成的仙人掌向左移动，当仙人掌到达屏幕最左端的时候消失，这里会结合removeChild()和setInterval();

第二个问题需要提前设定好一个boolean值，然后插入到每个函数即可。

练习遇到的困难：

  1. 绘制无限向左移动背景

这里结合了html5的canvas功能和js来实现，具体的原理就是动态绘制两个相同的图片，具体代码如下：


 <canvas id="background"></canvas>


//选取对应元素

const background = document.getElementById("background");

//确定绘制方法：

let ctx = background.getContext("2d");

//确定画布大小（在这个case里即整个背景板的大小）

background.width = 1000;

background.height = 200;

//用Image（）创建需要使用的图片

let img = new Image();

img.src = "t-rex-background.png";

//创建两张图片，确定两张图片左端起始点

function createBackground() {
let imgWidth = 1000;
let scrollSpeed = 8;
 function loop() {
    ctx.drawImage(img, imgWidth - 525, 0);
    ctx.drawImage(img, imgWidth - background.width, 0);
    imgWidth -= scrollSpeed;
if (imgWidth == 0) imgWidth = background.width;
//用requestAnimationFrame（）来实现滚动
window.requestAnimationFrame(loop);
}

  • 这里我遇到的主要问题是两个图片确实实现了向左移动，但是两个图片之间有空隙衔接不上，问题出现在我没有考虑到图片本身的大小，只考虑了容器的大小，当我把第一张图片最左端值修改为imgWidth-525（图片本身的宽度）问题就解决了。

  • 这个滚动实现的方法参考了这个博客：
    https://www.geeksforgeeks.org/html5-game-development-infinitely-scrolling-background/

  1. 仙人掌的生成和移动：

  • 这里的方法是先用document.createElement(“div”)创建一个新的div元素，然后再把这个div用appendChild（）的方法添加成为grid的子元素，再使用setTimeout在函数内部随机时间调用函数本身。

  • 这里我遇到的问题是一开始我不知道appendChild（）添加同一个名称是不会生成新的节点，所以一开始我的cactus是一个全局变量，导致代码运行无法实现我的需要的效果，但当把cactus变成函数内变量时，就可以实现我想要的效果了。
